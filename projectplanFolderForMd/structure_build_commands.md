# SousChef — PowerShell Build Commands

PowerShell commands that scaffold the exact monolith tree from [structure.md](structure.md): all folders
plus empty placeholder files. It is **idempotent and non-destructive** — it creates directories with
`-Force` and only creates a file if it doesn't already exist (so it never truncates your work).

## How to use
1. Open PowerShell in the parent directory where you want the project to live.
2. Paste the whole script below (or save it as `build.ps1` and run `./build.ps1`).
3. It creates `sous-chef/` with the full skeleton.

> **Not created here:** `.specify/` and `specs/`. Those are owned by SpecKit — `specify init` generates
> the `.specify/` scaffold (constitution, scripts, templates), and each `/speckit.specify` generates its
> own `specs/NNN-*/` folder with `spec.md` / `plan.md` / `tasks.md` using SpecKit's branch naming and
> numbering. Pre-creating them would leave stale, mis-numbered, or overwritten folders. Run SpecKit for
> those (see [sous-chef-spec-plan.md](sous-chef-spec-plan.md) §4).
>
> Other genuinely-empty folders get a `.gitkeep` so Git can track them. Files are created empty — fill
> them in via the SpecKit phases.

}

# 4) Create placeholder files (never overwrite existing content) ───────────────
$created = 0
foreach ($f in $files) {
  $path   = Join-Path $root $f
  $parent = Split-Path $path -Parent
  if (-not (Test-Path $parent)) { New-Item -ItemType Directory -Force -Path $parent | Out-Null }
  if (-not (Test-Path $path)) {
    New-Item -ItemType File -Path $path | Out-Null
    $created++
  }
}

Write-Host "Done. Skeleton at '$root/'  —  $($dirs.Count) dirs, $created new files." -ForegroundColor Green

## Verify the result
```powershell
# Show the tree (folders + files)
Get-ChildItem -Recurse sous-chef | Select-Object FullName

# Or a compact view of directories only
Get-ChildItem -Recurse -Directory sous-chef | Select-Object FullName
```

## Remove it (start over)
```powershell
Remove-Item -Recurse -Force sous-chef
```
